Alpha-Beta Pruning

.

May 20, 2018 • iBug

As described in a previous article, Alpha-Beta pruning can be used to speed up minimax heuristic searching by pruning branches that will never be reached.

A quick glance from Wikipedia:

It is an adversarial search algorithm used commonly for machine playing of two-player games (Tic-tac-toe, Chess, Go, etc.). It stops completely evaluating a move when at least one possibility has been found that proves the move to be worse than a previously examined move.

α-β pruning

The α-β pruning algorithm is based on the fact that each side wants to maximize his evaluated score, as well as to minimize the opponent’s score. So whenever one side finds a path with a score better than the worst score the other side can guarantee, we know that the other side will never let the game go to this path.

This image from Wikipedia demonstrates it well:

Let’s call the above tree “Levels 0 to 4”, where Level 0 contains only one node, the root node, and it’s where the searching starts. Levels 1 to 4 contains 3, 5, 7, 9 nodes each.

The tree is searched from left to right. It first completes the leftmost two nodes at level 4 and backtracks to the first node at level 2.

Greyed-out nodes are pruned by the α-β algorithm.

Heuristic searching

Here’s the pseudo-code in the previous post that demonstrates a plain heuristic searching algorithm.

function heuristicSearch(game, side, depth)
  if depth <= 0
    return null, heuristicScore(game)

  want_max = (side == BLACK)
  bestMove = null
  if want_max
    bestScore = -MAX - 1
  else
    bestScore = +MAX + 1

  for move in game.availableMoves
    game.perform move
    // Recurse
    subMove, subScore = heuristicSearch(game, side, depth - 1)
    game.undo

    if want_max
      if subScore > bestScore
        bestScore = subScore
        bestMove = move
    else
      if subScore < bestScore
        bestScore = subScore
        bestMove = move
  return bestMove, bestScore
